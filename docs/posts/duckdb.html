Summarize simulated data with duckdb

duckdb provides a high-performance to summarize massive data sets.

1 Introduction

This is a quick blog post showing how you can summarize a very large set of simulations using duckdb. You can learn more about duckdb here. The duckdb R package can take a long time to install, but will be worth it once your simulations grow out of control.

2 Setup

library(mrgsim.ds)
library(DBI)

3 Simulate

Simulate 1000 replicates.

unlink("temp-post", recursive = TRUE)

arrow::set_io_thread_count(2)

daemons(5, seed = 3579)

everywhere(library(mrgsim.ds))

out <- mirai_map(
  1:1000,
  \(x, mod, data) {
    mrgsim_ds(mod, data, tags = list(rep = x), recover = "dose")
  },
  .args = list(mod = mod, data = data)
)[.]

daemons(0)

out <- reduce_ds(out)

save_ds(out, "temp-post/duckdb.rds")

There are 4 billion rows in this simulation set across 1000 parquet files totaling 31 Gb of data on disk. It is important to note that the data are stored in highly-compressed parquet files; it’s only the simulation output object which provides access to tat data that is stored in .rds format. So the .rds file is very light.

Note that all your simulated data is now safely stored on disk. If your R session crashes or restarts for whatever reason, you can easily restart and pick up right where you left off without having to re-run the expensive simulation step.

4 Summarize using duckdb

Summarize using duckdb; we’ll take median, 5th and 95th percentiles of the Week 4 trough concentration by dose and replicate.

con <- dbConnect(duckdb())

ds <- as_arrow_ds(sims)

duckdb_register_arrow(con, "sims", ds)

system.time(
  
  summ <- dbGetQuery(con, "
    SELECT
      dose,
      time,
      rep, 
      COUNT(*)                     AS n,
      QUANTILE_CONT(IPRED, 0.025)  AS Lower,
      QUANTILE_CONT(IPRED, 0.5)    AS Median,
      QUANTILE_CONT(IPRED, 0.975)  AS Upper
    FROM sims
    WHERE time = 672
    GROUP BY time, dose, rep
    ORDER BY time, dose, rep
  ")

)

4.1 Summarize using duckdb pipeline

To do this, we work in the Apache Arrow R package and convert the arrow data set to a duckdb object first, without copying any data.

library(arrow)

system.time({ 

summ2 <- 
  ds %>% 
  to_duckdb() %>% 
  filter(time == 672) %>% 
  group_by(dose, time, rep) %>% 
  summarise(
    n = n(), 
    Lower = quantile(IPRED, 0.025, na.rm = TRUE), 
    Median = median(IPRED, na.rm = TRUE), 
    Upper = quantile(IPRED, 0.975, na.rm = TRUE), 
    .groups = "drop"
  ) %>% collect()
  
})
